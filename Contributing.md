# Contributing to ElderEase

Thank you for your interest in contributing to ElderEase 💙

We welcome contributions that help improve the project and create a better experience for everyone.

## Before You Start

Please make sure to:

* Check existing issues before creating a new one
* Avoid duplicate pull requests
* Read the project documentation carefully
* Follow the project structure and coding standards

---

## Contribution Scope

Currently, we are accepting contributions related to:

* UI/UX improvements
* Website responsiveness
* Bug fixes
* Documentation improvements
* Code optimization
* Accessibility enhancements

### Restricted/Internal Areas

The following areas are currently maintained internally by the core team:

* AI/ML systems
* Healthcare analytics logic
* Core backend architecture
* Database restructuring
* Deployment infrastructure
* Security-sensitive modules

Pull requests affecting these areas may not be accepted.

---

## How to Contribute

1. Fork the repository
2. Clone your fork
3. Create a new branch

```bash
git checkout -b feature-name
```

4. Make your changes
5. Commit your changes

```bash
git commit -m "Add: meaningful commit message"
```

6. Push to your fork

```bash
git push origin feature-name
```

7. Open a Pull Request

---

## Pull Request Guidelines

Please ensure that:

* Your code is clean and properly formatted
* The project builds/runs successfully
* Changes are relevant to the issue
* Large architectural changes are discussed first
* PR descriptions are clear and detailed

---

## Code of Conduct

Be respectful and constructive while interacting with other contributors.

Any form of harassment, hate speech, or inappropriate behavior will not be tolerated.

---

## Need Help?

If you have questions, feel free to open a discussion or issue.

Thank you for contributing to ElderEase 🚀
